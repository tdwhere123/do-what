# C005 - 收口为单入口启动

**优先级：** P0  
**依赖：** C004  
**涉及范围：**

- 根目录 `package.json`
- 如需新增脚本：`scripts/`
- `packages/app/src/main/`
- `packages/core/src/server/`

---

## 背景

当前主路径要求用户理解并手动执行多终端启动，这不符合 v0.1 封版收口的产品体验目标。  
在 C004 明确状态语义之后，应把启动主路径收口为单入口。

---

## 目标

1. 定义 `pnpm dev` 为默认启动入口。
2. 让单入口负责最小但可靠的启动编排。
3. 启动失败时给出清晰反馈，而不是静默等待或假成功。

---

## 本任务必须完成

1. 在根命令提供 `pnpm dev`。
2. 单入口至少负责：
   - 拉起 Core
   - 等待 Core 健康
   - Core 就绪后再启动 App
3. 不允许使用固定 `sleep` 代替健康检查。
4. Core 超时未就绪时，必须有清晰报错并退出。
5. `dev:core` 与 `dev:app` 继续保留，作为调试路径。

---

## 本任务不包含

- 不要求打包后的系统级自启动。
- 不要求一次解决所有平台差异。
- 不在本任务里同步修改 README 或实现状态文档。

---

## 验收标准（DoD）

1. 仓库根目录存在 `pnpm dev` 或等价的单入口命令。
2. 使用该命令可以自动完成 Core 与 App 的基本启动编排。
3. 单入口失败时有可解释报错，不是静默挂起。
4. `dev:core` / `dev:app` 仍可单独使用。
