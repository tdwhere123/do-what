# C006 - 建立 UI 交互分层模型

**优先级：** P0  
**依赖：** C005  
**涉及范围：**

- `packages/app/src/app/`
- `packages/app/src/components/`
- `packages/app/src/pages/`
- `UI/UI-DESIGN-SPEC.md`
- 如需交付物：`docs/archive/v0.1-closure/`

---

## 背景

当前 UI 的核心问题不只是少数按钮没接，而是没有统一的交互分层：

- 有些按钮应当真接线，但只是展示
- 有些按钮未接通，却被做成假业务入口
- 有些元素只是装饰，却被误认为操作按钮

在恢复页面结构之前，必须先建立统一的 A / B / C / D 模型。

---

## 目标

1. 让 Active、Empty、Settings 的关键交互全部被清晰分类。
2. 明确哪些动作必须真实可用，哪些只允许本地 UI，哪些必须诚实占位。
3. 为后续 C007-C012 的页面实现提供统一行为边界。

---

## 本任务必须完成

1. 以 `UI/UI-DESIGN-SPEC.md` 为准，建立 A / B / C / D 四类交互模型。
2. 覆盖至少以下页面：
   - Active / Workbench
   - Empty
   - Settings
3. 对每个关键交互写明：
   - 所属类别
   - v0.1 目标行为
   - 禁止的错误行为
4. 交付一份可检查的交互清单，供后续任务复用。
5. 分类结果必须成为 C012 占位硬化与 C013 最终 sign-off 的输入。

---

## 本任务不包含

- 不在本任务中完成全部视觉还原。
- 不要求一次把所有 C 类动作真正打通。
- 不要求一次完成所有占位按钮的最终 disabled 处理。

---

## 验收标准（DoD）

1. Active、Empty、Settings 的关键交互都有清晰分类。
2. 团队可以明确回答每个关键按钮是 A、B、C 还是 D。
3. 后续任务不再以“我理解成这样”为由自由发挥交互语义。
4. 分类结果可直接用于 C012 与 C013。
