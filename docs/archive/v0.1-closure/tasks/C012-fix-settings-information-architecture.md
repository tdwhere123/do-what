# C012 - 硬化占位能力与 UI 诚实性

**优先级：** P1  
**依赖：** C011  
**涉及范围：**

- `packages/app/src/components/`
- `packages/app/src/pages/`
- `packages/app/src/lib/commands/`
- `UI/UI-DESIGN-SPEC.md`
- `docs/archive/v0.1-closure/`

---

## 背景

在主路径、状态语义和主要页面都回正之后，剩余风险来自“看起来能用，但实际上没接通”的入口。  
这些入口若不统一硬化，会继续制造假提交、误点击和错误预期。

---

## 目标

1. 把所有未接通能力统一处理为诚实占位。
2. 以 C006 的 A / B / C / D 交互模型为准执行收口。
3. 消除会误导用户的假入口、假提交和空响应。

---

## 本任务必须完成

1. 所有 C 类能力必须统一处理为：
   - disabled
   - tooltip
   - `v0.2 实现`
   - 或纯展示
2. 覆盖至少以下区域：
   - Inspector
   - Settings
   - 历史入口
   - Soul 占位动作
   - 其他尚未接通但仍可见的入口
3. 未接通能力不能继续触发误导性命令、假提交或 desynced overlay。
4. D 类元素不得被误实现为业务按钮。

---

## 本任务不包含

- 不新增 v0.2 功能的真实后端链路。
- 不替代最终的视觉对照审查。
- 不重写主页面结构。

---

## 验收标准（DoD）

1. 所有未接通能力都有统一且诚实的占位处理。
2. 用户点击占位能力时，不再产生假效果或错误命令。
3. C006 的分类结果被真正落实到页面行为。
4. 页面不再混淆业务按钮与纯展示元素。

---

## 完成记录（2026-03-14）

- Empty 次动作 `浏览历史` 已改为 disabled 的 `v0.2` 诚实占位，并补充统一 tooltip / copy。
- Inspector 中未接通的治理与记忆维护动作继续保留为 disabled，可见但不会再触发误导性命令。
- 占位口径已收口到共享常量，避免多个页面继续漂移。
