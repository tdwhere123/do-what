# UI Style Bible（v0.6）

## 1. 设计目标

- 清晰：任务状态、权限、运行时信息一眼可见
- 稳定：交互层不因可选模块缺失而崩溃
- 可维护：统一 token、统一命名、统一页面骨架

## 2. 视觉原则

1. 面板分层清楚（主内容 > 次级面板 > 提示）
2. 运行时状态颜色固定且可读
3. 错误文案可执行（给出下一步命令）
4. 桌面与窄屏导航语义一致

## 3. 页面结构约定

- `Session`：消息流 + 上下文 + 工具轨迹
- `Dashboard`：模块入口与状态总览
- `Settings`：运行时、连接、调试、修复

## 4. 文案约定

- 避免抽象术语，优先动词 + 结果
- 错误提示必须包含：原因 + 下一步
- 对可选能力使用“未启用/可启用”，不要显示为“错误”

## 5. 可访问性基线

- 交互控件有可见 focus 状态
- 关键图标配文本或 tooltip
- 状态颜色需满足对比度

## 6. v0.6 特别规则

- Router 相关 UI 默认降级为“可选扩展”，不影响主流程
- 设置页必须清楚展示运行时可用性和缺失原因
